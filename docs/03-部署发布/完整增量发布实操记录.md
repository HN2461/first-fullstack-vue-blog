# 完整增量发布实操记录

本文档用于记录当前这种“前端改了、后端改了、数据库结构也可能动了”的完整增量发布过程。

它和《二次部署更新实操记录》的区别是：

- 二次部署更新：适合前端或后端小改，数据库没有结构风险。
- 完整增量发布：适合前端、后端、数据库相关逻辑都有变化，需要先备份、再发布、最后验证。

## 核心原则

本次发布按下面顺序执行：

```text
本地验证
  -> 本地打包前端
  -> 本地打包干净后端
  -> 上传发布包
  -> 服务器备份数据库、前端、后端、上传文件
  -> 发布前端
  -> 发布后端
  -> 视情况执行数据库迁移
  -> 重启后端
  -> 验证页面和接口
```

为什么要这样做：

- 先本地验证，是为了在上传前发现构建、测试、编码问题。
- 先备份服务器，是为了发布失败时可以回滚。
- 前端和后端分开打包，是因为它们是两个独立应用。
- 后端发布包不能包含 `.env`、`node_modules`、`uploads`，避免覆盖服务器环境配置和线上用户数据。
- 数据库结构变化不能直接覆盖线上库，除非明确要用本地数据库替换线上数据库。

### 本次中途问题 1：HTTP IP 登录页 importKey 报错

现象：

```text
Cannot read properties of undefined (reading 'importKey')
```

原因：

```text
当前通过 http://8.146.236.23 访问，属于非 HTTPS 的公网 IP。
浏览器在该环境下可能不提供 window.crypto.subtle。
旧前端登录流程强制调用 window.crypto.subtle.importKey，导致登录页报错。
```

处理：

```text
前端新增浏览器加密能力判断。
HTTPS / localhost / 支持 Web Crypto 时继续使用浏览器内密码加密。
HTTP IP / 不支持 crypto.subtle 时，临时走后端已有普通认证接口，并通过 Bearer token 维持登录态。
```

验证：

```text
重新构建并发布前端后，登录页可正常登录。
```

后续建议：

```text
正式生产环境应配置 HTTPS，不长期依赖 HTTP IP 兼容模式。
```

### 本次中途问题 2：RBAC 角色接口 pageSize=200 返回 400

现象：

```text
GET /api/rbac/roles?page=1&pageSize=200 400
```

原因：

```text
后端角色查询接口限制 pageSize 最大值为 100。
前端 collectAllPageItems 默认使用 pageSize=200，超过后端校验范围。
```

处理：

```text
将 frontend/src/services/admin.js 中 collectAllPageItems 默认 pageSize 从 200 调整为 100。
```

Copy-Item backend\package.json $release\
Copy-Item backend\package-lock.json $release\
Copy-Item backend\ecosystem.config.cjs $release\

### 目标

### 为什么要做

MongoDB 不强制表结构，但应用代码会依赖字段、类型、枚举和关系。如果旧数据不满足新代码预期，就需要迁移；如果新代码兼容旧数据，就不需要迁移。

### 判断方式

不需要迁移的常见情况：

- 只是新增非必填字段。
- 新字段有默认值。
- 页面或接口能处理字段为空。
- 旧数据不补字段也能正常展示。

需要迁移的常见情况：

- 字段改名。
- 字段类型变化。
- 旧枚举值不再被新代码识别。
- 新增必填字段，旧数据没有。
- 权限、角色、菜单结构变化。
- 文章、分类、媒体之间的关联方式变化。

## 第 19 步：发布后日志复查

### 目标

确认用户实际访问核心页面后，后端日志没有新的严重错误。

### 为什么要做

`/api/health` 只能证明服务启动成功，浏览器页面能打开也不代表所有接口都完全正常。日志可以发现接口参数错误、权限错误、旧数据字段不兼容、上传资源缺失等问题。

### 你现在要做什么

在服务器执行：

```bash
pm2 logs personal-blog-api --lines 120
```

该命令会持续监听日志，看完最近 120 行后按：

```text
Ctrl + C
```

退出日志查看。退出日志不会停止后端服务。

### 执行后怎么检查

重点判断：

- `GET/POST ... 200`：接口正常。
- `304`：浏览器缓存命中，通常正常。
- `401 /api/auth/me`：未登录时探测登录态，通常正常。
- `404 /uploads/...`：多半是历史上传文件缺失，记录下来后续单独处理。
- `400`：检查前端请求参数是否超过后端校验范围。
- `500`、`ValidationError`、`CastError`、`Unhandled`、数据库连接错误：阻塞发布，需要优先处理。

### 常见处理记录

如果看到：

```text
GET /api/rbac/roles?page=1&pageSize=200 400
```

说明前端全量拉取角色时 `pageSize` 超过后端允许值。优先把前端 `pageSize` 调整到后端允许范围，例如 `100`，重新构建并只发布前端。

如果看到：

```text
Cannot read properties of undefined (reading 'importKey')
```

这是浏览器登录页报错。常见原因是通过 `http://服务器IP` 访问时浏览器不提供 `window.crypto.subtle`。短期处理是发布支持 HTTP 试用环境的前端兼容逻辑；长期处理是配置 HTTPS。
