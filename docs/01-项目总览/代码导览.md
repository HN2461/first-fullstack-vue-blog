# 代码导览

这份文档用于快速理解当前代码结构。项目已经拆成两个独立应用，根目录不再承担 npm workspace 职责。

## 总体结构

```text
个人全栈博客系统/
├─ frontend/        # 前端应用，独立 package.json
├─ backend/         # 后端应用，独立 package.json
├─ deploy/          # Nginx / PM2 配置模板
├─ docs/            # 项目文档
├─ scripts/         # 仓库级辅助脚本，例如编码检查
└─ uploads/         # 本地上传文件目录
```

## 运行边界

- 前端只负责页面、路由、状态和 API 调用，构建产物是 `frontend/dist`。
- 后端只负责 API、认证、权限、数据库、上传和迁移脚本。
- MongoDB 是独立服务，后端通过 `backend/.env` 中的 `MONGODB_URI` 连接。
- 不再存在 `shared/` 运行时依赖；后端领域常量在 `backend/src/constants/domain.js`。

## 请求链路

```text
浏览器
  ↓
frontend 页面调用 /api/*
  ↓
开发环境：Vite proxy 转发到 backend
生产环境：Nginx 转发到 backend
  ↓
backend Express 路由
  ↓
service 业务逻辑
  ↓
Mongoose model
  ↓
MongoDB
```

## 本地开发

前端：

```bash
cd frontend
npm install
npm run dev
```

后端：

```bash
cd backend
npm install
npm run dev
```

默认端口：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:3001`
- MongoDB：`mongodb://127.0.0.1:27017/personal_fullstack_blog`

## 读代码建议

先看前端路由：`frontend/src/router/index.js`。

再看后端入口：`backend/src/app.js` 和 `backend/src/server.js`。

然后按功能读：

- 前端页面：`frontend/src/views`
- 前端接口：`frontend/src/services`
- 后端路由：`backend/src/routes`
- 后端业务：`backend/src/services`
- 后端模型：`backend/src/models`
