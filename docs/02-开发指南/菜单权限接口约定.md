# 菜单权限接口约定

## 菜单层级

系统菜单只保留两级：

- 一级菜单：顶层独立目录，例如「后台管理」「创作中心」。一级菜单 `parentType` 为 `root`，`parentId` 为 `null`，兼容字段 `parent_type` 为 `root`、`parent_id` 为 `0`。
- 二级菜单：挂载到某一个一级菜单下的功能入口。二级菜单 `parentType` 为 `child`，`parentId` 为所属一级菜单 id。

控制台渲染规则：

- 顶部导航只展示一级菜单。
- 左侧侧栏或移动端抽屉只展示当前一级菜单下的二级内容，不再重复展示一级菜单本身。
- 「知识库」是前端内置一级菜单，和后端返回的 RBAC 一级菜单平级展示。

## 后端接口

### 获取全部菜单

`GET /api/rbac/menus`

返回字段：

- `items`：扁平菜单列表。
- `tree` / `groups` / `roots`：按一级菜单分组的树结构，一级菜单节点下只包含直接二级菜单。

### 获取一级菜单

`GET /api/rbac/menus/roots`

返回字段：

- `items`：可展示的一级菜单列表，包含内置「知识库」和后端 RBAC 一级菜单。
- `maxRootMenus`：后端 RBAC 一级菜单数量上限，当前为 `10`。

### 创建或编辑菜单

`POST /api/rbac/menus`

`PATCH /api/rbac/menus/:id`

请求字段：

- `parentType` / `parent_type`：`root` 或 `child`。
- `parentId` / `parent_id`：二级菜单必须传所属一级菜单 id；一级菜单传 `null` 或 `0`。
- `name`、`code`、`icon`、`routePath`、`openMode`、`hidden`、`enabled`、`sortOrder`、`type`：常规菜单字段。

边界规则：

- 一级菜单最多 10 个，不包含前端内置「知识库」。
- 二级菜单只能挂载到一级菜单下，禁止继续嵌套三级。
- 内置 `management.root` 不允许删除。

### 菜单排序

`POST /api/rbac/menus/reorder`

排序只接受同层级调整：

- 一级菜单以 `parentId: null` 排序。
- 二级菜单以所属一级菜单 id 作为 `parentId` 排序。
- 二级菜单不能通过拖拽排序接口直接变成一级菜单。

## 前端约定

- `authStore.rootMenus` 是控制台菜单唯一入口来源。
- `ConsoleLayout` 顶部导航使用 `rootMenus`，左侧菜单使用当前 root 的 `children`。
- 控制台首页一级菜单卡片和快捷操作同样从 `rootMenus` 派生，新增一级菜单后无需额外写死入口。
