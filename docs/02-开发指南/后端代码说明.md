# 后端代码说明

后端是独立 Express + MongoDB 应用，目录为 `backend/`。

## 目录结构

```text
backend/
├─ src/
│  ├─ config/       # 环境、数据库、路由前缀
│  ├─ constants/    # 后端领域常量
│  ├─ middlewares/  # 鉴权、错误处理、请求指标
│  ├─ models/       # Mongoose 模型
│  ├─ routes/       # Express 路由
│  ├─ scripts/      # 管理员初始化、迁移、修复脚本
│  ├─ services/     # 业务逻辑
│  ├─ utils/        # 通用工具
│  └─ validators/   # Zod 参数校验
├─ tests/           # 后端测试
├─ uploads/         # 后端上传目录
├─ ecosystem.config.cjs
├─ package.json
└─ .env.example
```

## 启动链路

```text
src/server.js
  ↓
connectDatabase()
  ↓
createApp()
  ↓
app.listen(SERVER_PORT)
```

关键文件：

- `backend/src/server.js`：后端启动入口。
- `backend/src/app.js`：Express app、全局中间件、路由挂载。
- `backend/src/config/env.js`：读取 `backend/.env`。
- `backend/src/config/database.js`：MongoDB 连接。
- `backend/src/config/routes.js`：API 前缀，当前为 `/api`。

## 路由结构

后端路由统一挂在 `/api` 下：

- `/api/health`：健康检查
- `/api/auth`：注册、登录、当前用户、密码重置
- `/api/captcha`：验证码
- `/api/profile`：个人资料
- `/api/public`：公开门户数据
- `/api/admin`：管理后台
- `/api/comments`、`/api/articles/:id/like`、`/api/articles/:id/favorite`：互动接口

路由文件位于 `backend/src/routes`。

## 业务层

`routes` 只负责接收请求、校验和组织响应，主要业务逻辑放在 `backend/src/services`。

常见对应关系：

- `article.service.js`：文章管理
- `auth.service.js`：认证
- `category.service.js`：分类
- `comment.service.js`：评论
- `media.service.js`：媒体资源
- `search.service.js`：搜索
- `setting.service.js`：系统设置

## 数据模型

Mongoose model 位于 `backend/src/models`：

- `User.js`
- `Article.js`
- `Category.js`
- `Tag.js`
- `Comment.js`
- `Media.js`
- `Notification.js`
- `Reaction.js`
- `Setting.js`

后端领域常量位于：

```text
backend/src/constants/domain.js
```

例如用户角色、文章状态、评论状态。

## 环境变量

复制：

```bash
cd backend
cp .env.example .env
```

核心配置：

```env
NODE_ENV=development
SERVER_PORT=3001
CLIENT_ORIGIN=http://127.0.0.1:5173
MONGODB_URI=mongodb://127.0.0.1:27017/personal_fullstack_blog
JWT_SECRET=replace-with-a-long-random-secret
UPLOAD_DIR=uploads
```
