# 后端代码说明

后端是独立 Express + MongoDB 应用，目录为 `backend/`。

## 目录结构

```text
backend/
├─ src/
│  ├─ config/       # 环境、数据库、路由前缀
│  ├─ constants/    # 后端领域常量
│  ├─ middlewares/  # 鉴权、错误处理、请求指标
│  ├─ models/       # Mongoose 模型
│  ├─ routes/       # Express 路由
│  ├─ scripts/      # 管理员初始化、迁移、修复脚本
│  ├─ services/     # 业务逻辑
│  ├─ utils/        # 通用工具
│  └─ validators/   # Zod 参数校验
├─ tests/           # 后端测试
├─ uploads/         # 后端上传目录
├─ ecosystem.config.cjs
├─ package.json
└─ .env.example
```

## 新增后端文件放置规则

后端发布包默认会完整包含 `backend/src`，因此日常新增后端功能时，优先放到 `src` 下对应目录：

- 新增接口路由：放到 `backend/src/routes`。
- 新增业务逻辑：放到 `backend/src/services`。
- 新增 Mongoose 模型：放到 `backend/src/models`。
- 新增参数校验：放到 `backend/src/validators`。
- 新增领域常量：放到 `backend/src/constants`。
- 新增中间件：放到 `backend/src/middlewares`。
- 新增通用工具：放到 `backend/src/utils`。
- 新增一次性修复或迁移脚本：放到 `backend/src/scripts`。
- 新增后端测试：放到 `backend/tests`。

只要新增文件位于 `backend/src` 或 `backend/tests`，本地发布脚本会自动打进 `release/backend-release.zip`。

如果后续新增后端根目录下的运行必需目录，例如：

```text
backend/templates
backend/public
backend/views
backend/migrations
backend/prisma
backend/openapi
```

必须同步检查并更新 `scripts/package-release.ps1`，否则这些顶层目录不会自动进入后端发布包。

禁止把下面内容作为后端发布包内容：

```text
backend/.env
backend/node_modules
backend/uploads
backend/logs
backend/legacy-notes
```

原因：

- `.env` 是服务器生产环境配置，必须由服务器独立维护。
- `node_modules` 应在服务器本地重新安装。
- `uploads` 是线上上传数据，不能被本地目录覆盖。
- `logs`、`legacy-notes` 属于运行产物或迁移源资料，不属于常规后端代码发布内容。

## 启动链路

```text
src/server.js
  ↓
connectDatabase()
  ↓
createApp()
  ↓
app.listen(SERVER_PORT)
```

关键文件：

- `backend/src/server.js`：后端启动入口。
- `backend/src/app.js`：Express app、全局中间件、路由挂载。
- `backend/src/config/env.js`：读取 `backend/.env`。
- `backend/src/config/database.js`：MongoDB 连接。
- `backend/src/config/routes.js`：API 前缀，当前为 `/api`。

## 路由结构

后端路由统一挂在 `/api` 下：

- `/api/health`：健康检查
- `/api/auth`：注册、登录、当前用户、密码重置
- `/api/captcha`：验证码
- `/api/profile`：个人资料
- `/api/public`：公开门户数据
- `/api/admin`：管理后台
- `/api/comments`、`/api/articles/:id/like`、`/api/articles/:id/favorite`：互动接口

路由文件位于 `backend/src/routes`。

## 业务层

`routes` 只负责接收请求、校验和组织响应，主要业务逻辑放在 `backend/src/services`。

常见对应关系：

- `article.service.js`：文章管理
- `auth.service.js`：认证
- `category.service.js`：分类
- `comment.service.js`：评论
- `media.service.js`：媒体资源
- `search.service.js`：搜索
- `setting.service.js`：系统设置

## 数据模型

Mongoose model 位于 `backend/src/models`：

- `User.js`
- `Article.js`
- `Category.js`
- `Tag.js`
- `Comment.js`
- `Media.js`
- `Notification.js`
- `Reaction.js`
- `Setting.js`

后端领域常量位于：

```text
backend/src/constants/domain.js
```

例如用户角色、文章状态、评论状态。

## 环境变量

复制：

```bash
cd backend
cp .env.example .env
```

核心配置：

```env
NODE_ENV=development
SERVER_PORT=3001
CLIENT_ORIGIN=http://127.0.0.1:5173
MONGODB_URI=mongodb://127.0.0.1:27017/personal_fullstack_blog
JWT_SECRET=replace-with-a-long-random-secret
UPLOAD_DIR=uploads
```
