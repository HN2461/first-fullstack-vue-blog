# 项目记录台账导入格式

## 用途

项目记录台账支持通过 JSON 文件导入当天协作记录。该文件只记录当天完成、修复或调整的功能点，便于部署站点直接导入，避免为了同步台账记录迁移本地数据库。

## 文件位置

本地协作记录默认放在：

```text
backend/src/data/projectTimeline/daily/YYYY-MM-DD.json
```

例如：

```text
backend/src/data/projectTimeline/daily/2026-06-21.json
```

## JSON 结构

```json
{
  "schemaVersion": 1,
  "date": "2026-06-21",
  "source": "collaboration_daily",
  "records": [
    {
      "id": "project-timeline-import",
      "title": "新增项目记录文件导入能力",
      "detail": "新增项目记录台账导入入口，支持上传每日协作记录 JSON 文件。",
      "occurredAt": "2026-06-21T19:20:00+08:00",
      "category": "功能更新"
    }
  ]
}
```

## 字段说明

- `schemaVersion`：格式版本，当前固定为 `1`。
- `date`：记录所属日期，格式为 `YYYY-MM-DD`。
- `source`：记录来源，当前使用 `collaboration_daily`。
- `records`：当天协作记录列表，单个文件最多 100 条。
- `records[].id`：当天内唯一编号，建议使用英文短横线命名；导入时会和 `date` 组合成唯一标识，重复导入不会重复写入。
- `records[].title`：事件标题。
- `records[].detail`：事件详情，描述完成了什么、修复了什么、为什么调整。
- `records[].occurredAt`：事件时间，可选；不填时按当天中午时间兜底。
- `records[].category`：分类，支持 `内容上新`、`功能更新`、`问题修复`、`系统公告`、`部署发布`、`项目搭建`、`版本调整`、`手动记录`。

## 导入方式

进入后台：

```text
后台管理 / 系统运维 / 项目记录台账
```

点击 `导入记录`，选择当天 JSON 文件。后端会按 `source + date + records[].id` 幂等导入，已经存在的记录会被统计为“已存在”，不会重复创建。

## 维护规则

- 每完成一个明确功能、修复或重要调整，都应检查当天 JSON 是否已有对应记录。
- 项目记录不是碎片流水账；同一功能类、同一问题链路或同一协作主题应尽量合并成一条记录。
- 需要表达多个动作时，在同一条记录的 `detail` 中使用 `1.`、`2.`、`3.` 分点说明，不要拆成多条相近消息。
- 已有记录但描述不完整时，优先补充 `detail`。
- 只有确实属于不同功能类或不同协作主题时才新增一条记录。
- 当天文件只记录当天协作内容，不混入其他日期记录。
