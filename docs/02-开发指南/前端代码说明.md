# 前端代码说明

前端是独立的 Vue 3 + Vite 应用，目录为 `frontend/`。

## 目录结构

```text
frontend/
├─ src/
│  ├─ components/   # 复用组件
│  ├─ composables/  # 组合式逻辑
│  ├─ config/       # 前端配置
│  ├─ router/       # Vue Router
│  ├─ services/     # API 调用
│  ├─ stores/       # Pinia 状态
│  ├─ styles/       # 全局样式
│  ├─ utils/        # 工具函数
│  └─ views/        # 页面
├─ index.html
├─ vite.config.js
├─ package.json
└─ .env.example
```

## 入口文件

- `frontend/src/main.js`：创建 Vue 应用，注册 Pinia、Router、Ant Design Vue。
- `frontend/src/App.vue`：根组件，只承载 `router-view`。
- `frontend/src/router/index.js`：页面路由、登录守卫、角色访问控制。

## 页面分层

```text
frontend/src/views/
├─ public/     # 公开门户：首页、文章列表、文章详情、搜索
├─ auth/       # 登录、注册
├─ console/    # 登录后的知识库控制台
└─ admin/      # 管理功能页面
```

关键页面：

- `/`：公开门户首页
- `/login`：登录
- `/register`：注册
- `/console/articles`：知识库文章列表
- `/console/manage/articles`：文章管理
- `/console/manage/users`：用户管理
- `/console/manage/media`：媒体管理

## 接口调用

接口入口在 `frontend/src/services/http.js`。

默认同域请求 `/api/*`。如果前端和后端分开域名部署，可以在 `frontend/.env` 设置：

```env
VITE_API_BASE_URL=https://api.example.com
```

业务接口按模块拆分：

- `services/public.js`：公开页面接口
- `services/admin.js`：管理接口
- `services/interaction.js`：评论、点赞、收藏
- `services/http.js`：认证、个人资料、axios 实例

## 认证兼容规则

登录、注册、忘记密码、个人改密默认会在浏览器内使用 Web Crypto 对密码做一次性加密后提交。

注意：

- `https://`、`http://localhost`、`http://127.0.0.1` 通常支持 `window.crypto.subtle`。
- 直接通过公网 `http://服务器IP` 访问时，浏览器可能不提供 `window.crypto.subtle`。
- 试用服务器如果暂时没有 HTTPS，前端会降级使用后端兼容的普通认证接口，并通过 Bearer token 维持登录态。
- 正式生产环境应优先配置 HTTPS，不要长期依赖 HTTP IP 兼容模式。

如果登录页出现：

```text
Cannot read properties of undefined (reading 'importKey')
```

优先检查是否仍在使用旧前端包，或者是否通过非 HTTPS 的公网 IP 访问。

## 分页参数规则

后端部分列表接口会限制 `pageSize` 最大值，例如 RBAC 角色接口最大为 `100`。

前端拉取全量数据时，不要随意使用过大的 `pageSize`。通用全量分页默认使用：

```text
pageSize = 100
```

如果发布后日志出现类似：

```text
GET /api/rbac/roles?page=1&pageSize=200 400
```

说明前端请求参数超过后端校验范围，应优先把前端对应 pageSize 调整到后端允许值，而不是放宽后端校验。

## 状态管理

Pinia store 位于 `frontend/src/stores`：

- `auth.js`：登录用户、token、权限状态
- `app.js`：主题、布局等应用状态
- `notification.js`：公告/通知状态
- `site.js`：站点信息

## 后台列表页表格规范

后台业务列表页面统一使用 `frontend/src/components/BlogTable.vue`，不要在业务页面里直接拼装 Ant Design Vue 的分页表格。

统一布局规则：

- 页面外层使用固定工作区高度，例如 `height: calc(100vh - var(--console-header-height) - var(--console-content-padding) * 2)`，并设置 `overflow: hidden`，禁止页面整体因为列表行数产生纵向滚动。
- `BlogTable` 默认 `height="auto"`，组件会按自身距离视口底部的剩余空间计算固定高度；如果嵌入弹窗或特殊容器，可以显式传入 `height="560px"` 等固定高度。
- 空数据、少量数据、大量数据场景下，表格容器尺寸保持一致；分页栏固定在表格底部，不随数据行数上下移动。
- 列表数据超过表格主体可视范围时，只允许表格主体内部出现纵向滚动条；页面外层不承载列表滚动。
- 搜索、筛选、批量操作等控件统一放入 `BlogTable` 的 `toolbar` 插槽，与表格形成一体，不在表格外单独拆出筛选卡片或割裂式工具栏。
- 横向宽表通过 `scroll="{ x: 1000 }"` 或列 `width` 控制横向滚动；纵向滚动高度由 `BlogTable` 内部统一计算，业务页面不要手写 `scroll.y`。

## 构建

```bash
cd frontend
npm run build
```

产物在：

```text
frontend/dist
```

生产部署时只需要发布 `dist`。
