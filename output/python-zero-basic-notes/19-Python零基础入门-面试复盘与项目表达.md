# Python 零基础入门 19：面试复盘与项目表达

学完前面的内容，你已经不是“完全没接触过 Python”的状态了。

但企业面试不只看你背了多少概念，还会看你能不能把概念讲清楚，能不能用代码解决一个小问题。

这一篇专门做复盘。

## 你现在应该掌握什么

学完这套笔记后，至少要能说明：

- Python 代码如何运行
- 变量是什么
- 字符串、整数、小数、布尔值有什么区别
- `input()` 为什么得到字符串
- `if else` 怎么做条件判断
- `for` 和 `while` 怎么循环
- 列表和字典分别适合保存什么数据
- 函数为什么能减少重复
- 文件如何读写
- 异常处理有什么用
- 模块、包、pip、虚拟环境是什么
- 类、对象、属性、方法、self 是什么
- 常用标准库能解决哪些问题

这些就是 Python 基础面试的底盘。

## 高频问题一：列表和字典有什么区别

可以这样回答：

```text
列表适合保存一组有顺序的数据，比如多个商品名称、多个用户记录。
字典适合保存键值对，比如一个用户的姓名、年龄、城市。
实际项目里经常把它们组合起来，比如用列表保存多个用户字典。
```

示例代码：

```python
users = [
    {"name": "小明", "age": 18},
    {"name": "小红", "age": 20}
]

for user in users:
    print(user["name"])
```

## 高频问题二：函数有什么作用

可以这样回答：

```text
函数可以把一段可复用的逻辑封装起来，减少重复代码。
函数可以接收参数，也可以通过 return 返回结果。
在项目里，我会把独立职责的逻辑拆成函数，让代码更清晰。
```

示例代码：

```python
def calculate_total(price, count):
    return price * count

total = calculate_total(9.9, 3)
print(total)
```

## 高频问题三：print 和 return 有什么区别

可以这样回答：

```text
print 是把内容输出到屏幕，主要给人看。
return 是把函数结果返回给程序，方便后续继续计算或处理。
如果一个函数的结果还要被其他代码使用，就应该 return，而不是只 print。
```

## 高频问题四：什么是异常处理

可以这样回答：

```text
异常处理用于处理运行时可能出现的问题，比如输入格式错误、文件不存在、除数为 0。
我会用 try except 捕获具体异常，给用户友好提示，并避免程序直接崩溃。
不建议随便裸写 except，因为那会隐藏真正的错误原因。
```

## 高频问题五：为什么要用虚拟环境

可以这样回答：

```text
虚拟环境用于隔离不同项目的依赖。
不同项目可能需要不同版本的第三方库，如果都装在全局环境，容易产生版本冲突。
企业项目通常会配合 requirements.txt 记录依赖，方便别人安装和部署。
```

## 高频问题六：怎么理解面向对象

可以这样回答：

```text
面向对象是把业务里的事物抽象成类。
类定义属性和方法，对象是类创建出来的实例。
属性保存数据，方法表示行为。
这样可以把相关数据和操作放在一起，提高代码可维护性。
```

## 把记账本练习说成项目

不要只说：“我写了一个记账本。”

可以这样说：

```text
我做过一个命令行记账本练习。
它支持循环录入消费名称和金额，使用列表保存多条记录，使用字典保存单条记录。
录入结束后，程序会统计总支出，并把明细写入本地文本文件。
后续我还加入了异常处理，避免用户输入非数字金额时程序直接崩溃。
这个练习让我理解了输入、类型转换、循环、列表、字典、函数拆分和文件写入之间如何配合。
```

这段表达比“我会 Python 基础”更有说服力。

## 可以继续升级记账本

如果想把它变成更像企业项目的小作品，可以继续加：

- 用函数拆分录入、统计、保存逻辑
- 用异常处理校验金额
- 用 JSON 保存数据，而不是普通文本
- 支持读取历史记录
- 支持按日期统计
- 支持收入和支出分类
- 写一个 README 说明如何运行

## 升级版项目结构示例

```text
account_book/
  main.py
  storage.py
  calculator.py
  README.md
  requirements.txt
```

可以这样分工：

- `main.py` 负责程序入口和用户交互
- `storage.py` 负责读取和保存文件
- `calculator.py` 负责统计金额
- `README.md` 说明项目功能和运行方式
- `requirements.txt` 记录依赖

哪怕项目很小，只要结构清楚，面试表达就会更稳。

## 上手工作前的最低训练标准

如果目标不是“看懂教程”，而是“进入企业后能跟着做事”，建议至少完成下面这些训练。

### 训练一：能独立运行项目

你需要能做到：

- 拉到代码后知道先看 README。
- 知道创建虚拟环境。
- 知道安装依赖。
- 知道从入口文件启动程序。
- 看到报错时能读懂最后几行错误信息。

可以用下面流程练习：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python main.py
```

### 训练二：能读懂一个小功能

拿到一个小项目时，先找：

- 入口文件在哪里。
- 数据从哪里来。
- 经过哪些函数处理。
- 结果输出到哪里。

这就是最基础的数据流阅读能力。

### 训练三：能改一个小需求

以记账本为例，尝试完成这些需求：

- 新增收入记录。
- 支持按类型统计支出。
- 保存记录时增加日期字段。
- 查看汇总时显示最大一笔消费。
- 输入错误时不让程序崩溃。

这些需求都不大，但很接近真实工作里的“小改动”。

### 训练四：能写最小测试

工作中不能只靠手动运行。

比如有一个计算总金额的函数：

```python
def calculate_total(records):
    total = 0

    for record in records:
        total += record["amount"]

    return total
```

可以先用简单断言测试：

```python
records = [
    {"name": "早餐", "amount": 8},
    {"name": "地铁", "amount": 4}
]

assert calculate_total(records) == 12
```

如果没有报错，说明这个测试通过。

以后学习测试框架时，可以再用 `pytest` 写得更规范。

### 训练五：能写清楚变更说明

企业协作里，做完改动要能说明：

```text
本次修改：
1. 新增消费记录的金额校验，避免输入非数字导致程序崩溃。
2. 将记录保存格式从 txt 改为 JSON，方便后续读取和统计。
3. 拆分 load_records、save_records、show_summary 函数，降低 main 函数复杂度。

验证方式：
1. 手动添加一条消费记录。
2. 输入非法金额，确认程序提示重新输入。
3. 退出后重新运行，确认历史记录仍可读取。
```

这就是工作里常见的交付表达。

## 还缺什么才能真正投企业岗位

这套笔记能帮助你完成 Python 基础从零到一，也能让你开始做小脚本和基础项目练习。

但如果要正式投递企业岗位，还要按方向继续补技能：

- Python 后端：HTTP、FastAPI 或 Flask、数据库、接口鉴权、部署。
- 数据分析：pandas、Excel/CSV、数据清洗、图表、业务指标。
- 自动化测试：pytest、requests、接口测试、Web 自动化、测试报告。
- 爬虫与自动化：requests、BeautifulSoup、浏览器自动化、数据保存、反爬基础。

所以更准确地说：

```text
学完这套内容，可以具备 Python 入门基础、能写小工具、能开始读简单项目。
要达到企业独立开发，还需要继续完成一个方向的项目训练。
```

## 本篇小结

- 面试不是只背概念，要能结合代码和项目表达。
- 列表、字典、函数、异常、虚拟环境、面向对象都是基础高频点。
- 小项目也可以认真表达，重点是说明你用了什么、解决了什么问题、还能怎么改进。
- 学完这套笔记后，下一步要选择方向继续深入，并完成至少一个可运行、可说明、可验证的小项目。
