# Python 零基础入门 00：学习路线与运行方式

这套笔记是给"想从 0 到 1 学 Python，并且希望学完之后能尽快看懂公司项目"的读者准备的。

如果你已经写过一点前端 JavaScript，学习 Python 时很容易出现一种感觉：

> 这个东西我好像见过，但写法又不一样。

比如变量、字符串、数组/列表、对象/字典、函数、循环、条件判断，这些概念在 Python 和 JavaScript 里都存在。但是它们的语法习惯、运行方式、类型表现、项目组织方式又不完全一样。

更麻烦的是：如果你的 JS 本身也不算特别稳固，那学 Python 的时候就会出现"两个语言的概念互相打架"——你不确定某个写法到底是 Python 的还是 JS 的，也不敢确定 JS 那边原来是怎么写的。

所以后面的每一篇都会尽量做到三件事：

1. 先用小白能听懂的话解释 Python 当前知识点，讲清楚"它是什么、为什么要有它、怎么用"。
2. 再放一段 Python 代码，让你能复制运行，并逐行解释每一行在做什么。
3. 再把它和前端 JavaScript 对比，不是简单列个表格，而是把 JS 那边也讲清楚——告诉你哪里像、哪里不一样、哪里容易混、为什么会混。

你可以把这套笔记当成一套"双语对照学习材料"：主线是 Python，旁边顺手把 JS 也理一遍。

## 这套笔记怎么安排

建议按下面顺序阅读：

1. 学习路线与运行方式

但是更推荐按顺序学，因为前面的概念会在后面不断复用。比如你不理解变量，后面看函数、列表、字典都会卡；你不理解条件判断，后面做项目练习也会卡。

## 学完这套笔记能达到什么程度

如果你把每篇代码都亲手运行一遍，并完成每篇的小练习，应该能达到下面几个结果：

1. 能看懂 Python 基础代码，不再害怕变量、条件、循环、列表、字典、函数这些核心概念。
2. 能写出简单但完整的小程序，比如记账本、文件整理、数据统计脚本。
3. 能看懂公司项目里常见的基础 Python 写法，比如函数调用、配置读取、异常处理、模块导入。
4. 面试时能用自己的话说明基础语法、异常处理、模块环境、面向对象和项目练习，而不是只背概念。
5. 能把 Python 和 JavaScript 的相似点、差异点分清楚，不再脑子里来回打架。
6. 顺便把 JS 里面本来就不太稳固的知识也重新理清，因为你会在对比中更理解"这个概念本质是什么"。

需要注意的是：这套笔记定位是"Python 基础从零到一 + 项目入门准备"。如果目标是正式投递 Python 后端、数据分析、自动化测试岗位，还需要继续补对应方向的框架、数据库、网络请求、测试、项目部署等内容。

## Python 和 JS 学习时最大的区别

如果你学过前端，你对 JavaScript 的第一印象通常是：

- 它运行在浏览器里。
- 它可以操作页面（DOM）。
- 它经常和 HTML、CSS 一起出现。
- 它写完之后，刷新浏览器就能看到效果。

而 Python 的第一印象通常是：

- 它更多运行在命令行、脚本、服务端、数据处理环境里。
- 它不直接操作浏览器页面。
- 它经常用来处理文件、数据、接口、自动化任务。
- 它通常通过 `python 文件名.py` 来运行。

可以先这样理解：

| 对比项 | Python | JavaScript |
| --- | --- | --- |
| 常见运行位置 | 电脑命令行、服务器、脚本任务 | 浏览器、Node.js、前端项目 |
| 常见用途 | 后端接口、脚本、数据处理、自动化、AI | 页面交互、前端工程、Node.js 后端 |
| 文件后缀 | `.py` | `.js` / `.ts` |
| 输出文字 | `print()` | `console.log()` |
| 代码块标识 | 缩进（4 个空格） | `{}` 大括号 |
| 语句结尾 | 一般不写分号 | 可写可不写分号，前端项目常按规范统一 |
| 变量声明 | 直接写变量名赋值，不需要关键字 | 需要 `let`、`const`、`var` |
| 注释 | `#` 单行，`"""..."""` 多行 | `//` 单行，`/* ... */` 多行 |
| 命名习惯 | 下划线：`user_name` | 小驼峰：`userName` |

最容易混的地方是：**概念相似，但语法不一样。**

比如输出一句话：

```python
print("你好，Python")
```

JavaScript 里通常写：

```js
console.log('你好，JavaScript')
```

这两段代码做的事情很像，都是把内容输出到某个地方。但在实际开发中：

- Python 的 `print()` 常见于命令行脚本、调试、学习代码。
- JS 的 `console.log()` 常见于浏览器控制台、Node.js 控制台、前端调试。

再比如声明变量：

```python
name = "小明"
```

```js
const name = '小明'
```

Python 直接写 `变量名 = 值`，不需要任何声明关键字。而 JS 需要 `let` 或 `const` 来声明。如果你 JS 基础也不太稳固，这里先记住：`const` 声明后不能重新赋值，`let` 声明后可以重新赋值。Python 里没有这个区分，变量随时可以重新赋值。

## 什么是"运行代码"

写代码只是把命令写在文件里。

运行代码，就是让电脑照着这些命令做事。

这个概念对前端开发者来说可能需要特别想一下，因为前端开发的日常流程是"写代码 → 保存 → 浏览器自动刷新 → 看效果"，你几乎不会手动去"运行"一段 JS 代码。浏览器的 V8 引擎在帮你运行。

而 Python 初学阶段，你需要自己手动运行代码。

比如下面这行 Python 代码：

```python
print("你好，Python")
```

意思是：让电脑在屏幕上显示一句话。

运行结果：

```text
你好，Python
```

如果换成 JavaScript：

```js
console.log('你好，JavaScript')
```

在浏览器控制台或 Node.js 里运行，结果通常是：

```text
你好，JavaScript
```

你可以把它们对应起来记：

| 你想做的事 | Python 写法 | JS 写法 |
| --- | --- | --- |
| 输出一句话 | `print("内容")` | `console.log('内容')` |
| 写一行注释 | `# 注释` | `// 注释` |
| 保存 Python 文件 | `demo.py` | 不适用 |
| 保存 JS 文件 | 不适用 | `demo.js` |

### "运行"到底发生了什么

不管 Python 还是 JS，运行代码的本质是一样的：你写的文字（源代码）被一个"翻译程序"翻译成电脑能理解的指令，然后电脑执行这些指令。

只是不同语言的"翻译程序"不同：

| 语言 | "翻译程序" | 说明 |
| --- | --- | --- |
| Python | Python 解释器 | 你安装 Python 后就自带的程序，逐行读取 `.py` 文件并执行 |
| JS（浏览器） | 浏览器的 V8 等引擎 | 浏览器内置，打开网页时自动运行 |
| JS（Node.js） | Node.js 运行时 | 需要单独安装，用 `node 文件名.js` 运行 |

所以当你写 `python hello.py` 的时候，实际是在说："请 Python 解释器来读这个文件，然后照着做。"

这和你前端项目里 `npm run dev` 的本质是一样的——只是 `npm run dev` 背后启动了一个开发服务器，自动帮你做了很多事，你不用自己手动跑 `node` 命令。

## Python 代码一般怎么运行

Python 最常见的运行方式是：

```bash
python 文件名.py
```

比如你新建一个文件：

```text
hello.py
```

文件内容是：

```python
print("我正在学习 Python")
```

在命令行里进入这个文件所在目录，然后运行：

```bash
python hello.py
```

如果环境正常，你会看到：

```text
我正在学习 Python
```

这和前端开发里运行 JS 的感觉不太一样。

前端项目里你可能经常写：

```bash
npm run dev
```

然后打开浏览器看页面。Python 初学阶段很多时候不需要启动一个完整项目，只需要运行一个 `.py` 文件就行。

也可以用 Node.js 来类比：

```bash
# 运行 Python 文件
python hello.py

# 运行 JS 文件（用 Node.js）
node hello.js
```

这两者的感觉几乎一样：命令行里敲一个命令，然后看到输出。

可以这样对照：

| 场景 | Python | 前端 JS |
| --- | --- | --- |
| 运行单个学习文件 | `python hello.py` | `node hello.js` 或浏览器控制台 |
| 运行项目 | `python app.py`、框架命令、脚本命令 | `npm run dev` |
| 安装依赖 | `pip install 包名` | `npm install 包名` |
| 依赖记录 | `requirements.txt`、`pyproject.toml` | `package.json`、`package-lock.json` |
| 包管理工具 | pip | npm / yarn / pnpm |

现在不用急着记住所有东西。你只需要先记住：Python 入门时，重点是能把 `.py` 文件跑起来。

### 交互式运行（REPL）

除了写 `.py` 文件再运行，Python 还有一种特别方便的学习方式：交互式运行。

在命令行里直接输入 `python`（不带文件名），会进入一个交互模式：

```text
>>> print("你好")
你好
>>> 1 + 2
3
>>> name = "小明"
>>> name
'小明'
```

`>>>` 是 Python 交互模式的提示符，表示"请输入代码"。你输入一行，Python 立刻执行一行并显示结果。

这个模式特别适合学习时快速试代码，不用每次都建文件。

JS 也有类似的东西：在浏览器里按 F12 打开控制台，或者用 Node.js 输入 `node` 进入交互模式：

```text
> console.log('你好')
你好
> 1 + 2
3
```

退出方式：

| 语言 | 进入交互模式 | 退出交互模式 |
| --- | --- | --- |
| Python | 输入 `python` | 输入 `exit()` 或按 `Ctrl+Z` 回车（Windows）/ `Ctrl+D`（Mac/Linux） |
| Node.js | 输入 `node` | 按两次 `Ctrl+C` 或输入 `.exit` |

## 每篇文章应该怎么学

建议你每篇都按这五步学。

### 第一步：先看概念

不要一上来背语法。先理解这节内容是解决什么问题。比如变量是为了给数据起名字，函数是为了把一段逻辑打包复用。

如果你 JS 也不太稳固，正好趁机重新理解这个概念的本质。比如"变量"在 Python 和 JS 里做的事情是一样的——给数据起个名字，方便后面用。只是写法不同。

### 第二步：复制 Python 代码运行

不要只看。哪怕代码很简单，也要复制到本地运行一次。

比如：

```python
print("今天开始学习 Python")
```

### 第三步：把代码改一点再运行

比如改成：

```python
print("我正在练习第一段 Python 代码")
```

只要运行结果跟着变化，就说明你不是在背文章，而是在真正控制代码。

### 第四步：看 JS 对照

比如 Python 里：

```python
print("今天开始学习 Python")
```

JS 里：

```js
console.log('今天复习 JavaScript')
```

你要观察的不是"哪个更好"，而是：

- 它们解决的问题是不是类似？
- 写法哪里不一样？
- 以后写项目时会不会混？

如果你对 JS 那边的写法也不确定，不要跳过——停下来，把 JS 的也搞清楚。这样两个语言你都不会再糊。

### 第五步：写一条自己的总结

比如学完输出，你可以写：

```text
Python 用 print 输出内容，JS 常用 console.log 输出内容。它们都能调试和查看结果，但 Python 更常出现在命令行脚本里，JS 更常出现在浏览器控制台或前端项目里。
```

这种总结比死背概念有用很多。

## 不要把 Python 缩进和 JS 大括号混在一起

这是你后面最容易头疼的地方之一。

### JS 用大括号 `{}` 表示代码块

JavaScript 里代码块通常用 `{}` 包起来：

```js
if (score >= 60) {
  console.log('及格')
}
```

解释一下这段 JS：

- `if (score >= 60)` —— 条件判断，小括号里是条件
- `{` —— 代码块开始
- `console.log('及格')` —— 条件成立时执行的代码
- `}` —— 代码块结束

大括号的作用就是：**告诉电脑"这些代码是一组的，属于 if 条件成立时要执行的"。**

### Python 用缩进表示代码块

Python 不用 `{}` 表示代码块，而是用冒号和缩进：

```python
score = 80

if score >= 60:
    print("及格")
```

解释一下这段 Python：

- `if score >= 60:` —— 条件判断，注意没有小括号，末尾有冒号 `:`
- `    print("及格")` —— 缩进了 4 个空格，表示它"属于"if 里面的代码

缩进的作用就是：**告诉电脑"这行缩进了，所以它属于上面那个 if 的代码块"。**

### 为什么这容易混

因为两段代码表达的逻辑完全一样，但"代码块怎么表示"这件事差异很大。

如果你 JS 也不太稳固，这里再把 JS 的大括号说清楚：

**JS 大括号不只是 if 才用**，很多地方都用：

```js
// 函数也用大括号
function sayHello() {
  console.log('你好')
}

// for 循环也用大括号
for (let i = 0; i < 3; i++) {
  console.log(i)
}
```

**Python 对应的写法全部用缩进**：

```python
# 函数用缩进
def say_hello():
    print("你好")

# for 循环也用缩进
for i in range(3):
    print(i)
```

### Python 缩进不对会怎样

Python 对缩进非常敏感。下面这样写就不对：

```python
score = 80

if score >= 60:
print("及格")
```

运行后会报错：`IndentationError: expected an indented block`。

因为 `print("及格")` 没有缩进，Python 不知道它是不是属于 `if` 里面的代码。

更危险的混淆是这样：

```python
score = 80

if score >= 60:
    print("及格")
print("不管怎样都执行")
```

最后一行 `print("不管怎样都执行")` 没有缩进，所以它**不属于 if 代码块**，不管分数多少都会执行。

如果你本来想让两行都在 if 成立时执行，应该写成：

```python
score = 80

if score >= 60:
    print("及格")
    print("不管怎样都执行")  # 这行也缩进了，属于 if 代码块
```

对比 JS 中的等价写法：

```js
const score = 80

// 第一段：只有第一行属于 if
if (score >= 60) {
  console.log('及格')
}
console.log('不管怎样都执行') // 不在大括号里，不属于 if

// 第二段：两行都属于 if
if (score >= 60) {
  console.log('及格')
  console.log('不管怎样都执行') // 在大括号里，属于 if
}
```

你可以先记住一句话：

> JS 用大括号划分代码块，Python 用缩进划分代码块。大括号看得见，缩进看不见但同样重要。

这句话非常重要。后面讲条件判断、循环、函数、类的时候都会反复用到。

### 缩进规则总结

| 规则 | Python | JS |
| --- | --- | --- |
| 怎么表示代码块 | 缩进（推荐 4 个空格） | `{}` 大括号 |
| 条件语句末尾 | 冒号 `:` | 无特殊标记 |
| 缩进不对会怎样 | 直接报错 `IndentationError` | 不影响运行，但影响代码可读性 |
| 缩进多少 | 必须统一（4 个空格是标准） | 不强制，但通常 2 个空格 |
| 能混用 Tab 和空格吗 | **不能**，会报错 | 不会报错，但不推荐 |

## 不要把 Python 的列表和 JS 的数组完全混为一谈

Python 里有 `list`，前端 JS 里有 `Array`。

它们都能存一组数据。

Python：

```python
names = ["小明", "小红", "小刚"]
print(names[0])
```

JS：

```js
const names = ['小明', '小红', '小刚']
console.log(names[0])
```

输出的都是第一个元素：

```text
小明
```

看起来很像对吧？但后面深入之后你会发现，它们的方法名、常见操作、语言习惯不完全一样。

比如添加元素：

```python
names.append("小丽")    # Python 用 append
```

```js
names.push('小丽')      // JS 用 push
```

再比如获取长度：

```python
len(names)    # Python 用 len() 函数
```

```js
names.length  // JS 用 .length 属性
```

所以学习时不要只记"Python list 就是 JS Array"。更准确的说法是：

> Python 的 list 和 JS 的 Array 都是顺序集合，很多使用场景相似，但方法名和语言习惯不同。

后面第 10 篇会详细展开。现在先建立这个意识：**它们像，但不是完全同一个东西，不能无脑套用。**

## 不要把 Python 的字典和 JS 的对象完全混为一谈

Python 里有 `dict`，JS 里有对象 `{}`。

它们都能保存"键和值"的关系。

Python：

```python
user = {
    "name": "小明",
    "age": 18
}

print(user["name"])
```

JS：

```js
const user = {
  name: '小明',
  age: 18
}

console.log(user.name)  // 或者 user['name']
```

它们看起来很像，但也有区别：

| 对比项 | Python dict | JS object |
| --- | --- | --- |
| 常见中文叫法 | 字典 | 对象 |
| 取值常见写法 | `user["name"]` | `user.name` 或 `user['name']` |
| key 是否常写引号 | 字符串 key 通常写引号 | 普通标识符 key 可不写引号 |
| 遍历方式 | `for key in user:` | `for (const key in user)` 或 `Object.keys(user)` |
| 常见用途 | 配置、接口数据、映射关系 | 页面状态、接口数据、组件 props、配置 |

这里特别说明一下 key 的引号问题，这是新手最容易混的：

**Python 字典的 key 通常是字符串，所以写引号：**

```python
user = {
    "name": "小明",   # key 是字符串 "name"
    "age": 18         # key 是字符串 "age"
}
```

**JS 对象的 key 如果是合法标识符，可以不写引号：**

```js
const user = {
  name: '小明',   // name 不写引号，JS 自动当成字符串
  age: 18         // age 不写引号
}
```

这不是语法错误，是两个语言的习惯不同。Python 那边 key 几乎都写引号，JS 那边 key 不写引号更常见。

后面讲字典时会展开说。现在只需要先建立感觉：它们像，但不是完全同一个东西。

## 新手最常见的误区

### 误区一：一开始就想全部理解

刚开始学编程，不需要每个词都理解到很深。

先做到"能看懂大概意思，能跑起来"，后面再慢慢补细节。

你学 JS 的时候也一样。第一次见 `const`、`let`、函数、数组、对象时，不可能一次全懂。Python 也是这样。

### 误区二：只看不敲

编程是动手技能。只看文章会觉得懂了，但真正复制、运行、修改时，才会暴露问题。

尤其是 Python 的缩进、英文符号、文件路径、命令行运行方式，必须亲手试，光看很容易觉得简单，动手时却卡住。

### 误区三：一报错就觉得自己不适合

报错不是失败，而是电脑在告诉你："这里我没看懂。"

比如少写一个引号：

```python
print("你好)
```

程序就会报错：`SyntaxError: EOL while scanning string literal`。

修正成下面这样就可以：

```python
print("你好")
```

JS 里也一样：

```js
console.log('你好)
```

这也会报错：`SyntaxError: Invalid or unexpected token`。修成：

```js
console.log('你好')
```

你会发现，很多报错不是你"不会编程"，只是符号、拼写、缩进、括号这些细节错了。

### 误区四：看到相似语法就直接套用

这是从 JS 转 Python 时特别容易出现的问题。

比如你可能习惯写：

```js
if (age >= 18) {
  console.log('成年人')
}
```

然后到了 Python，不能照着写成：

```python
if (age >= 18) {
    print("成年人")
}
```

Python 正确写法是：

```python
age = 18

if age >= 18:
    print("成年人")
```

区别在哪：

| 部分 | JS | Python |
| --- | --- | --- |
| 条件要不要小括号 | 要：`if (条件)` | 不要：`if 条件` |
| 条件后面 | `{` | `:` 冒号 |
| 代码块 | `{ ... }` | 缩进 |
| 比较运算符 | `>=`、`===`、`!==` | `>=`、`==`、`!=`（没有 `===`） |

所以后面每篇文章都要提醒自己：概念可以迁移，语法要按 Python 来写。

### 误区五：两个语言的概念互相干扰

这是最隐蔽的误区。不是"不会"，而是"以为自己会，但把另一个语言的写法混进来了"。

典型例子：

1. **数组方法串混**：在 Python 里写 `names.push("小丽")`（应该用 `append`）
2. **对象取值串混**：在 Python 里写 `user.name`（字典通常用 `user["name"]`）
3. **分号串混**：在 Python 里每行结尾写 `;`（不报错但不符合习惯）
4. **声明关键字串混**：在 Python 里写 `let name = "小明"` 或 `const name = "小明"`（Python 不需要声明关键字）
5. **逻辑运算符串混**：在 Python 里写 `&&` 或 `||`（Python 用 `and` 和 `or`）
6. **注释串混**：在 Python 里写 `// 注释`（应该用 `#`）

这些错误不会让你"不会编程"，但会让你写出来的代码跑不动或看起来别扭。

后面每篇都会特别标注"容易和 JS 混淆"的地方，帮你逐步把这些区分清楚。


比如学到变量时，你可以建：

```text
04_variables.py
```

里面写：

```python
name = "小明"
age = 18

print(name)
print(age)
```

前端 JS 如果做同样练习，可能是：

```js
const name = '小明'
const age = 18

console.log(name)
console.log(age)
```

两个都能表达"保存名字和年龄，然后输出"。但：

- Python 用 `print()`，JS 用 `console.log()`
- Python 变量赋值不需要 `let` 或 `const`，JS 通常需要
- Python 没有声明关键字，直接写 `name = "小明"` 就行

如果你对 JS 的 `let` 和 `const` 区别也不太清楚，这里补一句：

- `const` 声明的变量不能再重新赋值（比如 `const age = 18; age = 20` 会报错）
- `let` 声明的变量可以重新赋值（比如 `let age = 18; age = 20` 没问题）
- Python 里没有这个区分，所有变量都可以随时重新赋值

## 你需要记住的第一句话

学习编程不是比谁记得快，而是比谁更愿意：

- 复制运行
- 看报错
- 改代码
- 再运行
- 对比差异
- 写自己的总结

Python 和 JS 放在一起学时，最重要的是不要急着判断"哪个更简单"。

你要做的是建立一张对照表：

| 我想做的事 | Python 怎么写 | JS 怎么写 |
| --- | --- | --- |
| 输出内容 | `print()` | `console.log()` |
| 写注释 | `#` | `//` |
| 声明变量 | `name = "小明"` | `const name = '小明'` 或 `let name = '小明'` |
| 保存一组数据 | `list`（列表） | `Array`（数组） |
| 保存不可变数据 | `tuple`（元组） | 无原生类型 |
| 保存键值关系 | `dict`（字典） | `object`（对象） |
| 去重和集合运算 | `set`（集合） | `Set` |
| 定义函数 | `def` | `function` / 箭头函数 |
| 条件判断 | `if 条件:` | `if (条件) {` |
| 循环 | `for x in 列表:` | `for (const x of 数组) {` |
| 处理异常 | `try except` | `try catch` |
| 导入模块 | `import` | `import` / `require` |
| 逻辑与/或 | `and` / `or` | `&&` / `||` |
| 比较相等 | `==`（值相等）/ `is`（同一对象） | `==`（隐式转换）/ `===`（严格相等） |
| 不等于 | `!=` | `!==` |

后面每篇文章都会把这张表一点点填细。

## 本篇小结

- 这套笔记主线是 Python，但每篇都会加入 JavaScript 对照，两个语言都会讲清楚。
- Python 和 JS 有很多相似概念，比如变量、字符串、列表/数组、字典/对象、函数、循环。
- 相似不代表完全一样，学习时要把"概念迁移"和"语法差异"分开。
- Python 入门时最重要的是能运行 `.py` 文件，能看懂输出，能根据报错修改代码。

- Python 缩进不对会直接报错，不像 JS 只是"不好看"。
- 两个语言概念互相干扰是最隐蔽的学习障碍，后面每篇都会标注"容易和 JS 混淆"的地方。
- 学习方法是：看解释、跑代码、改代码、看 JS 对照、写自己的总结。
