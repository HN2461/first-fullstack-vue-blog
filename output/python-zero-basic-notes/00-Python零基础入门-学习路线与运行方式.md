# Python 零基础入门 00：学习路线与运行方式

这套笔记是给“完全没写过程序”的读者准备的。

你不需要先懂计算机专业词汇，也不需要背很多概念。学习 Python 最好的方式是：**先看懂一句代码，再复制运行，再改一点点试试看。**

## 学完这套笔记能达到什么程度

如果你把每篇代码都亲手运行一遍，并完成每篇的小练习，应该能达到三个结果：

- 能看懂 Python 基础代码，不再害怕变量、条件、循环、列表、字典、函数这些核心概念。
- 能写出简单但完整的小程序，比如记账本、文件整理、数据统计脚本。
- 面试时能用自己的话说明基础语法、异常处理、模块环境、面向对象和项目练习，而不是只背概念。

需要注意的是：这套笔记定位是“Python 基础从零到一 + 面试入门准备”。如果目标是正式投递 Python 后端、数据分析、自动化测试岗位，还需要继续补对应方向的框架、数据库、网络请求、测试、项目部署等内容。

## 学习时准备什么

你只需要准备：

- 一台电脑
- Python 解释器
- 一个能写代码的编辑器，比如 VS Code

如果你暂时还没有安装环境，也可以先阅读概念，后面再统一实操。

## 什么是“运行代码”

写代码只是把命令写在文件里。

运行代码，就是让电脑照着这些命令做事。

比如下面这行代码：

```python
print("你好，Python")
```

意思是：让电脑在屏幕上显示一句话。

运行结果：

```text
你好，Python
```

## 建议的学习方法

每篇文章按这四步学：

1. 先看文字解释，不急着背。
2. 复制代码运行一次。
3. 改一个小地方再运行一次。
4. 记录自己看不懂或报错的地方。

举个例子：

```python
print("今天开始学习 Python")
```

你可以改成：

```python
print("我正在练习第一段代码")
```

只要运行结果跟着变化，就说明你已经开始真正使用 Python 了。

## 新手最常见的误区

### 误区一：一开始就想全部理解

刚开始学编程，不需要每个词都理解到很深。

先做到“能看懂大概意思，能跑起来”，后面再慢慢补细节。

### 误区二：只看不敲

编程是动手技能。只看文章会觉得懂了，但真正复制、运行、修改时，才会暴露问题。

### 误区三：一报错就觉得自己不适合

报错不是失败，而是电脑在告诉你：“这里我没看懂。”

比如少写一个引号：

```python
print("你好)
```

程序就会报错。修正成下面这样就可以：

```python
print("你好")
```

## 你需要记住的第一句话

学习编程不是比谁记得快，而是比谁更愿意：

- 复制运行
- 看报错
- 改代码
- 再运行

这就是程序员每天都在做的事。
