# Python 零基础入门 01：Python 是什么

Python 是一种编程语言。

你可以把编程语言理解成：**人和电脑沟通时使用的一套指令格式。**

人平时说中文、英文。电脑不能直接理解这些自然语言，所以我们用 Python、JavaScript 这样的编程语言，把想让电脑做的事情写清楚。

如果你接触过前端 JavaScript，可以先这样理解：

- JavaScript 常用于告诉浏览器：“页面上这个按钮被点击后要做什么。”
- Python 常用于告诉电脑或服务器：“帮我处理数据、文件、接口、脚本任务。”

它们都是编程语言，都能表达逻辑，但常见工作场景不一样。

## 先用一句话理解 Python

Python 是一种语法相对清晰、适合快速写脚本、后端接口、自动化任务、数据处理和 AI 相关开发的编程语言。

这句话可以拆开看：

1. **语法相对清晰**：代码读起来比较接近自然表达，新手容易入门。
2. **适合快速写脚本**：比如批量整理文件、批量重命名、读取表格。
3. **适合后端接口**：公司项目里可以用 Python 写 API，给前端页面提供数据。
4. **适合自动化任务**：比如定时爬取公开数据、自动生成报表、自动处理日志。
5. **适合数据和 AI**：很多数据分析、机器学习、人工智能库都优先支持 Python。

你不需要一开始就学完这些方向。刚开始只要先掌握基础语法，后面才能进入具体方向。

## Python 和 JavaScript 都是什么关系

Python 和 JavaScript 是两种不同的编程语言。

它们不是谁替代谁，而是常用于不同位置。

| 对比项 | Python | JavaScript |
| --- | --- | --- |
| 常见角色 | 脚本语言、后端语言、数据处理语言 | 前端页面交互语言，也可做 Node.js 后端 |
| 常见运行环境 | Python 解释器、服务器、命令行 | 浏览器、Node.js |
| 最常见文件后缀 | `.py` | `.js` / `.ts` |
| 初学输出方式 | `print()` | `console.log()` |
| 页面交互 | 不是主要用途 | 核心用途之一 |
| 数据处理 | 很常见 | 也能做，但生态侧重点不同 |
| 后端开发 | Django、Flask、FastAPI 等 | Express、NestJS、Koa 等 |

如果你已经学过前端，可以把它想成：

> JavaScript 更像你和浏览器页面沟通的语言；Python 更像你和电脑任务、服务器逻辑、数据处理流程沟通的语言。

这个说法不是绝对的，因为 JS 也能写后端，Python 也能做一些图形界面或 Web 页面相关事情。但对入门阶段来说，这样理解最容易建立方向感。

## Python 能做什么

Python 可以做很多事情。

### 1. 自动整理文件

比如你下载了一堆图片、文档、压缩包，希望按后缀分类。

Python 可以写脚本扫描文件夹，然后把文件移动到不同目录。

这一类任务在前端 JS 里不是不能做，但浏览器里的 JS 不能随便操作你电脑上的文件系统。Node.js 可以操作文件系统，但对很多非前端岗位来说，Python 写这种脚本更常见。

### 2. 处理 Excel 表格

比如公司给你一批 Excel，需要统计金额、筛选数据、生成新表。

Python 有很多成熟库可以处理表格数据。

前端 JS 也能处理 Excel，比如在后台管理系统里上传 Excel、解析 Excel、导出 Excel。但如果是本地批量处理、数据清洗、自动生成报表，Python 的使用场景非常多。

### 3. 写网站后端

前端页面需要从后端拿数据。

比如文章列表页面要请求接口：

```text
GET /api/articles
```

这个接口可以用很多语言写，Python 就是其中一种。

Python 后端可以接收请求、查询数据库、处理业务逻辑，然后返回 JSON 给前端。

前端 JavaScript 负责把这些 JSON 数据渲染成页面。

可以这样分工：

| 工作 | Python 后端 | 前端 JS |
| --- | --- | --- |
| 查数据库 | 常见 | 通常不直接做 |
| 写接口 | 常见 | Node.js 可以做，浏览器 JS 不做 |
| 渲染页面 | 不是主要工作 | 核心工作 |
| 处理按钮点击 | 不是主要工作 | 核心工作 |
| 返回 JSON | 常见 | 请求并使用 JSON |

### 4. 做数据分析

Python 在数据分析里非常常见。

比如：

- 统计销售数据
- 分析用户行为
- 清洗脏数据
- 画图表
- 训练模型前处理数据

JS 也能做图表展示，比如 ECharts、D3、AntV。但 JS 更多出现在“把图表展示到页面上”；Python 更多出现在“把数据算出来、整理好、分析好”。

### 5. 写爬虫采集公开数据

Python 经常用于请求网页、解析内容、保存数据。

这里要注意：爬虫必须遵守网站规则、法律法规和 robots 协议，不是任何内容都能随便抓。

JS 也可以请求接口、解析页面，但 Python 的爬虫资料和库非常多，新手更容易找到教程。

### 6. 做人工智能相关开发

很多 AI、机器学习、深度学习相关工具链都大量使用 Python。

比如你可能听过：

- NumPy
- Pandas
- PyTorch
- TensorFlow
- scikit-learn

现在不用急着学这些。先学基础语法，后面才能看懂这些库的示例代码。

## 为什么 Python 适合入门

Python 的优点是：代码看起来比较接近日常表达。

比如让电脑显示一句话：

```python
print("你好，Python")
```

你大概能猜到：`print` 是“打印、输出”的意思，括号里是要显示的内容。

JavaScript 里通常写：

```js
console.log('你好，JavaScript')
```

这两段代码做的事情类似，都是输出一段文字。

区别是：

- Python 用 `print()`。
- JS 常用 `console.log()`。
- Python 字符串可以用双引号，也可以用单引号。
- JS 字符串也可以用单引号、双引号或模板字符串。

从入门体验看，Python 少了很多前端初学时容易干扰注意力的内容。

前端新手经常要同时面对：

- HTML 结构
- CSS 样式
- JavaScript 逻辑
- 浏览器兼容
- DOM 操作
- 构建工具

Python 入门时可以先只面对：

- 一个 `.py` 文件
- 几行代码
- 命令行运行结果

这就是 Python 适合入门的重要原因：你可以更专注地理解“代码逻辑”本身。

## Python 不是万能的

Python 很适合入门，也很实用，但它不是所有场景的最佳选择。

比如：

- 写浏览器里的交互页面，通常会用 JavaScript。
- 写大型高性能游戏引擎，通常不会首选 Python。
- 写手机原生 App，通常会用其他语言。
- 写极致性能要求的底层系统，通常会用 C、C++、Rust、Go 等。

所以不要把 Python 理解成“学了它就什么都不用学”。

更合理的理解是：

> Python 是一门非常适合建立编程思维、处理实际任务、进入后端/数据/自动化/AI 方向的语言。

如果你已经会一点 JS，那 Python 会帮你把“编程逻辑”从“页面交互”扩展到“脚本任务、服务端和数据处理”。

## 第一个感受：命令从上往下执行

看下面的 Python 代码：

```python
print("第一行")
print("第二行")
print("第三行")
```

运行结果：

```text
第一行
第二行
第三行
```

Python 默认会从上往下执行代码。

这就像你写了一张任务清单：

1. 先显示“第一行”
2. 再显示“第二行”
3. 最后显示“第三行”

电脑会按照顺序做。

JavaScript 也一样，普通同步代码也是从上往下执行：

```js
console.log('第一行')
console.log('第二行')
console.log('第三行')
```

输出：

```text
第一行
第二行
第三行
```

这里你可以先记住一个共同点：

> 不管是 Python 还是 JS，最基础的代码执行顺序都是从上到下。

后面 JS 会遇到异步，比如 `setTimeout`、`Promise`、`async/await`。Python 也有异步，但入门阶段先不用急。先把顺序执行、条件、循环、函数这些基础掌握好。

## 代码里的符号要用英文符号

新手很容易把中文标点写进代码里。

正确写法：

```python
print("你好")
```

错误写法：

```python
print（"你好"）
```

第二段里括号是中文全角括号，Python 看不懂。

JS 里也一样，不能写中文括号：

```js
console.log（'你好'）
```

正确写法是：

```js
console.log('你好')
```

写代码时，括号、引号、逗号、冒号，建议都使用英文输入法下的符号。

这个习惯非常重要，因为很多新手报错不是概念不懂，而是符号输错了。

## Python 文件和 JS 文件有什么不同

Python 文件一般以 `.py` 结尾。

比如：

```text
hello.py
```

里面写：

```python
print("你好，Python")
```

运行方式通常是：

```bash
python hello.py
```

JS 文件一般以 `.js` 结尾。

比如：

```text
hello.js
```

里面写：

```js
console.log('你好，JavaScript')
```

如果用 Node.js 运行，命令通常是：

```bash
node hello.js
```

如果是在浏览器里，也可以写在 HTML 页面引入的脚本里，或者直接在浏览器控制台运行。

对照一下：

| 文件 | 内容示例 | 运行方式 |
| --- | --- | --- |
| `hello.py` | `print("你好")` | `python hello.py` |
| `hello.js` | `console.log('你好')` | `node hello.js` 或浏览器控制台 |

## Python 代码读起来为什么“像英文”

Python 很多语法词都比较直观。

比如：

```python
if age >= 18:
    print("成年人")
```

里面的 `if` 就是“如果”。

再比如：

```python
for name in names:
    print(name)
```

读起来像：

```text
对于 names 里的每一个 name，把它打印出来。
```

JS 也有类似概念：

```js
for (const name of names) {
  console.log(name)
}
```

两者表达的逻辑相似，但写法不同：

- Python 用 `for name in names:`。
- JS 用 `for (const name of names) {}`。
- Python 用缩进表示循环内部代码。
- JS 用大括号表示循环内部代码。

这就是后面学习时最重要的原则：**概念可以对照，语法要分开记。**

## 本篇小练习

第一步，新建一个 Python 文件：

```text
what_is_python.py
```

写入：

```python
print("Python 是一种编程语言")
print("它可以写脚本、后端、数据处理和自动化任务")
print("我会把它和 JavaScript 对照起来学习")
```

运行后应该看到：

```text
Python 是一种编程语言
它可以写脚本、后端、数据处理和自动化任务
我会把它和 JavaScript 对照起来学习
```

第二步，如果你有 Node.js，新建一个 JS 文件：

```text
what_is_js.js
```

写入：

```js
console.log('JavaScript 是一种编程语言')
console.log('它常用于前端页面交互，也可以用 Node.js 写后端')
console.log('我会观察它和 Python 的相同点与不同点')
```

第三步，自己补一句总结：

```text
Python 和 JavaScript 都能表达程序逻辑，但 Python 更常用于脚本、后端和数据处理；JavaScript 更常用于浏览器页面交互和前端工程。
```

这句话你可以按自己的理解改写。能说清楚，比背原文更重要。

## 本篇小结

- Python 是一种编程语言，用来让人把任务用电脑能理解的方式表达出来。
- Python 常用于脚本、后端、数据处理、自动化、AI 等方向。
- JavaScript 常用于前端页面交互，也可以通过 Node.js 写后端。
- Python 和 JS 都能表达变量、条件、循环、函数等逻辑，但语法和常见场景不同。
- Python 入门常用 `print()` 输出内容，JS 常用 `console.log()` 输出内容。
- Python 代码和 JS 代码都要使用英文符号，中文括号、中文引号很容易导致报错。
- 学 Python 时可以借助 JS 旧知识理解概念，但不要直接把 JS 语法套进 Python。
